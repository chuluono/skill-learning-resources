# 贡献指南

感谢您对技能知识自学资源平台的关注和支持！我们欢迎并鼓励社区成员贡献内容，共同构建这个可以学习任何东西的开源平台。

## 贡献方式

您可以通过以下方式为项目做出贡献：

1. **提交新内容**：添加新的技能领域、学习资源或实践案例
2. **改进现有内容**：更新过时信息、修正错误或增强内容质量
3. **提供反馈**：通过 Issue 报告问题、提出建议或讨论改进方案
4. **完善文档**：改进项目文档和使用指南
5. **社区支持**：回答其他用户的问题，帮助维护社区秩序

## 贡献流程

### 1. 提交 Issue

在开始贡献前，建议先查看现有 Issue，确保您的贡献不会与他人的工作重复。如果没有相关 Issue，请创建一个新的 Issue，描述您计划贡献的内容：

- **Bug 报告**：详细描述问题、复现步骤和期望行为
- **功能请求**：说明您希望添加的内容及其价值
- **内容改进**：描述您计划改进的内容和原因

### 2. 准备贡献

1. **Fork 仓库**：在 GitHub 上 Fork 项目仓库到您的个人账号
2. **克隆仓库**：将 Fork 后的仓库克隆到本地
   ```bash
   git clone https://github.com/your-username/skill-learning-resources.git
   cd skill-learning-resources
   ```
3. **创建分支**：为您的贡献创建一个新的分支
   ```bash
   git checkout -b feature/your-feature-name
   ```

### 3. 贡献内容

请确保您的贡献符合以下规范：

#### 内容规范

- **准确性**：确保信息准确、最新，引用可靠来源
- **全面性**：覆盖技能的主要方面和应用场景
- **实用性**：提供可操作的学习资源和方法
- **清晰性**：结构清晰，使用一致的格式和语言风格
- **客观性**：公正评价不同资源的优缺点，避免主观偏见
- **原创性**：尊重知识产权，引用他人内容时注明来源

#### 格式规范

- 使用 Markdown 格式编写所有内容
- 遵循项目现有的目录结构和文件命名规范
- 使用一致的标题层级和列表格式
- 为资源添加适当的标签和分类
- 提供清晰的文件路径和链接

### 4. 提交 Pull Request

1. **提交更改**：将您的更改提交到本地分支
   ```bash
   git add .
   git commit -m "Add: 简明描述您的贡献"
   ```

2. **推送分支**：将本地分支推送到您的 GitHub 仓库
   ```bash
   git push origin feature/your-feature-name
   ```

3. **创建 PR**：在 GitHub 上创建一个新的 Pull Request
   - 选择适当的基础分支（通常是 `main` 或 `master`）
   - 填写详细的 PR 描述，说明您的贡献内容和目的
   - 引用相关的 Issue 编号（如果有）

### 5. 审核流程

1. **PR 审核**：项目维护者将审核您的 PR，可能会提出修改建议
2. **修改反馈**：根据审核意见进行必要的修改
3. **合并 PR**：审核通过后，您的 PR 将被合并到主分支

## 内容贡献指南

### 添加新技能

1. **确定分类**：确定技能所属的领域和学科
2. **创建目录**：在相应的学科目录下创建技能目录
3. **编写内容**：按照以下结构创建内容：
   - `README.md`：技能概述、学习路径图
   - `resources/`：推荐学习资源
   - `learning-path/`：结构化学习路径
   - `methods/`：学习方法指导
   - `examples/`：实践案例

### 改进现有内容

1. **识别问题**：确定需要改进的内容和原因
2. **进行修改**：更新内容，确保准确性和一致性
3. **验证更改**：检查修改是否符合项目规范

### 资源推荐标准

推荐资源时，请考虑以下因素：

- **质量**：内容质量高，讲解清晰
- **相关性**：与技能直接相关，针对性强
- **可访问性**：易于获取，考虑不同地区用户的访问权限
- **多样性**：提供多种类型的资源（书籍、课程、文章等）
- **更新时间**：优先推荐较新的资源，注明资源的发布或更新时间
- **难度级别**：明确标注资源的难度级别（入门、中级、高级）

## 常见问题

### Q: 我可以贡献非技术领域的内容吗？

**A:** 当然可以！本项目旨在覆盖所有领域的技能知识，包括艺术、商业、科学等非技术领域。

### Q: 如何确定技能的分类？

**A:** 请参考项目现有的分类体系，选择最适合的领域和学科。如果您认为需要添加新的分类，请先创建 Issue 讨论。

### Q: 我可以推荐付费资源吗？

**A:** 可以，但请同时提供一些免费的替代资源，并明确标注资源的付费性质。

### Q: 如何确保我推荐的资源质量？

**A:** 建议您亲自使用或研究过这些资源，或参考可靠的评价和推荐。同时，请注明资源的适用场景和优缺点。
